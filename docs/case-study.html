AI PM Case Study

I built FlatFeed to turn fragmented Berlin WBS apartment listings into a trusted, measurable matching workflow.

FlatFeed collects apartment listings through source adapters, normalizes messy listing text into one catalog, matches listings to user filters, and uses AI QA to review parser quality without letting AI mutate user-facing results.

01

The Problem

Finding WBS-eligible apartments in Berlin is fragmented and time-sensitive: listings appear across different sources, use inconsistent formats, change quickly, and often hide key eligibility details inside unstructured text. The affected user is a renter who must repeatedly check multiple catalogs and manually interpret WBS, district, rent, room count, and availability before they can even decide whether an apartment is worth opening.

  • Multiple listing sources
  • Inconsistent housing text
  • High manual review burden
02

Why AI?

I did not make AI responsible for matching because eligibility decisions need to be predictable and explainable. I used deterministic parsing and rules for the user-facing flow, then used AI as a controlled QA layer to review parser snapshots, flag ambiguous fields, and improve quality over time. AI is valuable here because listing text has edge cases and wording variation that are expensive to review manually, while the product still keeps AI away from automatic data mutation.

  • Rules for matching
  • AI for review
  • Humans decide feedback
03

My Role

I defined the product scope, shaped the portfolio positioning, designed the source-collection and matching flow, implemented the prototype, created the synthetic evaluation dataset, wrote deterministic parsing rules, added AI QA with budget controls, and built the admin dashboard for review and measurement.

  • Product definition
  • Prototype delivery
  • Evaluation design
04

The Approach

I scoped FlatFeed around a trusted catalog rather than a generic real-estate search tool. The data strategy uses synthetic Berlin listings with hidden ground truth so the parser can be evaluated without scraping or redistributing real housing-company listings. I chose deterministic parsing for fields that affect matching, a source-adapter ingestion layer for future multi-source collection, SQLite for a local prototype, and AI QA only as an admin-reviewed control layer.

Collect Source adapters pull listings into one pipeline.
Normalize Parser extracts WBS, rent, rooms, floor, address, district.
Match Deterministic rules compare listings with saved filters.
Review AI QA flags risky parser snapshots for admin feedback.
Notify Users receive active matches once, with source context.
05

What I Built

I built a Telegram bot and Streamlit dashboard prototype. The bot stores a fixed user filter for WBS, Berlin district, maximum Kaltmiete, and rooms, then returns active listings that match. The ingestion layer normalizes source listings into a shared schema, enriches local transit walking-time estimates, tracks source activity, deduplicates sent notifications, and records source health.

Results

In the current synthetic golden-set eval, FlatFeed parses 15 synthetic listings with 100.0% parser field accuracy and 100.0% exact listing accuracy across WBS, rent, rooms, floor, district, postal code, and special constraints. These are synthetic evaluation metrics, not production user-impact numbers, but they demonstrate a measurable quality loop instead of an anecdotal demo.

07

What I Learned

The biggest lesson is that AI PM work is strongest when the AI boundary is clear. For this product, deterministic rules create trust for user-facing matching, while AI adds value as a review and learning layer. If I continued the project, I would add a small set of live source adapters where terms allow it, expand the golden set, add screenshots from a real demo session, and compare AI QA prompt versions over time with human-reviewed feedback.

Determinism first Rules own user-facing eligibility decisions.
AI as a control AI reviews quality without silently changing data.
Measure honestly Demo metrics are useful when they are clearly labeled.
